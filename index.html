Tambah Perangkat

Router / Access Point khusus untuk perangkat DI SISI PELANGGAN (mis. router WiFi rumah). "Router / MikroTik" tetap khusus perangkat di server/kantor saja — jangan dicampur supaya Dashboard Monitoring & Live Interface tidak salah baca.
HTB tetap harus 1 set berpasangan (1 pemancar + 1 penerima). Dari HTB, boleh diteruskan paralel lewat Switch/Hub sebagai anaknya, ATAU langsung ke satu pelanggan lewat kolom "Pelanggan Terhubung" di bawah — dua-duanya sah, tidak wajib salah satu.
Satu port di perangkat induk yang sama hanya boleh dipakai satu perangkat anak — port yang sudah terpakai otomatis ditandai & tidak bisa dipilih dobel.
Pakai ini kalau perangkat ini adalah titik akhir ke SATU pelanggan tertentu (mis. HTB atau router di rumah pelanggan) — tidak perlu bikin perangkat tambahan cuma untuk "mewakili" pelanggan. Untuk ODP yang dipakai banyak pelanggan sekaligus, biarkan kosong di sini.
Isi HANYA kalau perangkat ini adalah router MikroTik itu sendiri. Daftar di bawah diambil LANGSUNG dari router (hanya port ether fisik) setelah server terhubung — kalau belum terkoneksi, tes koneksi dulu di halaman Pengaturan. Kalau dipilih, 2 kolom trafik di bawah akan otomatis diperbarui terus oleh agent tiap ±20 detik — TIDAK bisa diketik manual lagi selama field ini terisi. Untuk HTB/splitter pasif yang bukan MikroTik, biarkan kosong.
Untuk HTB/splitter/hub: kalau sudah ada pelanggan yang di-link ke node ini atau turunannya (field "Pelanggan Terhubung" di atas), trafik di diagram topologi OTOMATIS dari jumlah trafik pelanggan-pelanggan itu — 2 kolom manual di atas hanya jadi fallback selama belum ada pelanggan yang di-link.
Buka lokasi di Google Maps → tekan lama titiknya → salin koordinat yang muncul → tempel di sini. Tetap bisa isi manual di kolom Latitude/Longitude, atau geser pin di peta seperti biasa — tidak wajib pakai peta lagi.
Posisi di peta belum ditentukan — akan ditempatkan di tengah peta, geser untuk mengatur lokasi sebenarnya.

Buat Pengumuman

Tambah Integrasi

⚠️ Jangan simpan API key, token, atau secret apa pun di kolom ini — kolom ini bisa dibaca oleh siapa pun yang punya akses server ini.

Tambah Pelanggan

Kalau username ini sudah ada sebagai PPP Secret di MikroTik, cukup isi persis sama — sistem akan menautkannya (bukan membuat ulang) begitu siklus live agent berjalan.

Tambah Server / MikroTik

Isi HANYA alamat/domainnya saja — jangan sertakan port di sini (mis. kalau layanan tunnel Anda memberi "namahost:3627", tulis "namahost" di sini dan "3627" di kolom Port API di bawah). Kalau port ikut tertulis di sini, sistem akan otomatis memisahkannya untuk Anda.
Write-only — tidak pernah ditampilkan balik ke browser setelah disimpan, dan hanya bisa dibaca oleh agent sentral di VPS Anda (lewat service_role key, bukan browser). Disimpan sebagai teks di database (bukan terenkripsi) sehingga akses dibatasi murni lewat Row Level Security — hanya admin yang bisa melihat/mengubah baris ini.

Ganti WiFi Device

Perintah dikirim ke antrean acs_commands dan dieksekusi agent dalam beberapa detik (device harus online / mendukung connection request).
